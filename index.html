HLSE305 - Restitution du travail fait en TP

Cette page permet d'expliquer comment utiliser le site au cas où vous auriez des problèmes, mais permet aussi d'expliquer quelles ont été les technologies utilisées pour la réalisation de ce site car, en effet, plusieurs outils bien utiles m'ont aidé.

Naviguer

Naviguer entre les pages

Afin de naviguer aisément sur le site, il suffit de glisser sa souris à gauche de l'écran (là où se trouvent les flèches), pour faire apparaître les différents TPs faits durant ce semestre.
Survoler un de ces titres permet de faire dérouler à son tour tous les documents réalisés durant celui-ci.

Naviguer dans la page

Le sous-menu est celui qui est présent lorsque que vous ne mettez pas votre souris à gauche de l'écran, il permet de naviguer vers les différents chapitres composants chaque pages. De plus, le chapitre que vous êtes actuellement en train de parcourir se met en évidence afin que vous ne vous perdiez jamais.

Structure du site

La structure du site est de la forme suivante :

  • TP 14
    • css
      • fichier.css
    • fichier.html
    • document_response.pdf
    • second_fichier.html
  • TP 92
    • diapo.odp
  • TP 12
    • doc.tex
    • presentation.pdf
C'est à dire que chaque TP réalisé tout au long du semestre possède son propre dossier dans lequel on peut retrouver chaque document s'y rattachant.

Git

Qu'est-ce que Git ?

Git est un gestionnaire de version très connu et énormément utilisé dans le domaine de la programmation. Il permet de gérer les modification de son code en donnant à chacune d'elles un numéro de version.
De plus, il permet de créer ce que l'on appelle des branches afin de travailler à plusieurs sur le même code sans pour autant se géner l'un et l'autre, il suffit juste d'effectuer une action appelée merge (fusionner en français) afin de regrouper les deux codes écrits par l'un et par l'autre. Pour une documention complète je vous conseille de renseigner sur ce site.

Ce site est en ligne !

Ce projet est possède justement un dépôt Git en ligne que vous pouvez consulter à cette adresse, vous pourrez ainsi voir l'évolution du code au travers du temps et comprendre plus facilement l'architecture du site. De plus ce dépôt possède une page GitHub permettant donc de voir ce site depuis n'importe où grâce à une connexion internet.

Less

Qu'est-ce que le Less

Le Less est un langage se rapprochant beaucoup du css, en effet celui-ci se veut être un clone du css avec une synthaxe plus souple et surtout possédant des fonctionnalité supplémentaire. Si vous voulez vous renseigner sur le Less, n'hésitez pas à consulter la documentation de ce dernier.
Ci-dessous je vous présente quelques une des fonctionnalités que le Less propose, en espérant que vous réfléchirez à l'adopter pour vos future projet car celui-ci permet une meilleur organisation de fichiers et une synthaxe bien plus claire.
Pour résumer, le Less est tout simplement une amélioration du CSS qui se transpile en CSS.

Les classes imbriquées

Les classes imbriqués permettent de simplifer grandement la synthaxe mais rendre le code beaucoup plus clair. Pour démontrer la puissance de cette fonctionnalité je vais présenter deux code qui donnent le même résultat, l'un sera en css, l'autre utilisera le Less.

fichier.css 
            
p
{
    background-color: blue;
}

p:hover
{
    background-color: red;
}

p a
{
    text-decoration: none;
}

p a:hover
{
    text-decoration: underline;
}
fichier.less 
            
p
{
    background-color: blue;

    &:hover
    {
        background-color: red;
    }

    a
    {
        text-decoration: none;

        &:hover
        {
            text-decoration: underline;
        }
    }

}

Comme nous pouvons le voir & fait référence à l'élément actuel, cela permet d'éviter de séparer des styles s'appliquants sur un seul et même élément. De même, on peut voir qu'il est plus simple de suivre la hiérarchie des styles avec le Less bien qu'ici l'exemple soit vraiment simple. Ici l'utilisation du Less ne permet pas de réduire le nombre de lignes écrites, cependant nous verrons plus loin comment certaines fonctionnalités permettent d'obtenir un code plus court.

Les variables

Less permet l'ajout de variables dans notre feuille de style, cela peut être très utile lorsque nous voulons utiliser une même valeur à plusieurs endroits dans notre fichier. En effet, si nous venons un jour à modifier cette valeur nous n'aurons pas besoin de chercher de partout où nous l'avous utilisé car celle-ci sera une variable.
De plus nous avons la possibilité de faire des calcules avec ces variables (additions, soustractions, etc ...).
Ici encore, je vais donner un exemple de ce que pourrait donner un code CSS converti en Less.

fichier.css 
            
p strong
{
    color: red;
    background-color: blue; /* Doit être la couleur du texte du paragraphe */
}
p
{
    color: blue; /* Couleur du texte du paragraphe */
    background-color: orange;
}
fichier.less 
            
@paragraph-text-color: blue;

p
{
    color: @paragraph-text-color;
    background-color: orange;

    strong
    {
        color: red;
        background-color: @paragraph-text-color;
    }
}

Ici il suffit de changer la valeur de la variable que nous avons défini grâce au prefix @ afin de changer la couleur correspondante tout en gardant les contraintes voulues. Plus besoin de relire tout le code pour chercher où l'attribut doit être changé.
Nous allons nous arrêter ici pour le Less, mais n'hésitez surtout pas à faire un tour du côté de la documention pour découvrir toutes les fonctionnalités proposées.

JavaScript

Introduction

Le JavaScript permet de dynamiser un site, on l'utilise pour animer (lorsque le css ne nous permet pas le faire) mais aussi pour ajoter dynamiquement du contenu au site ou encore bien d'autres choses. Sur ce site j'utilise le JavaScript pour remplir le sous-menu, je cherche en premier lieu tous les titre h2 se trouvant sur la page puis ajoute un lien vers ces derniers dans le sous-menu. Nous allons voir que le JavaScript est un élément central des sites web de nos jours.

Les Frameworks et les dérivés du langage

Il existe différens frameworks JavaScript, très connus, tels que AngularJS, Ajax et bien d'autres. Ils permettent de manipuler les éléments de la page web plus facilement mais aussi ajoutent des fonctionnalités permettants de communiquer avec le serveur web de manière efficace.
Certains langages ont aussi dérivés sur JavaScript tels que TypeScript, PureScript, etc ... ils permettent par exemple d'avoir un langage fortement typé, une synthaxe plus courtes.
Je vous met ci-dessous une petite conférence faite lors de Devoxx 2015 (oui, c'est vieux) qui explique plus que correctement les concepts de JavaScript.

Si vous avez regardé la conférence, JavaScript ne devrait plus avoir de secrets pour vous !
Comme vous avez pu le voir plus haut dans la partie sur le Less, j'utilise un script JavaScript pour la coloration synthaxique du code, ce dernier est très connu et s'appelle highlightjs.
